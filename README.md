# Miyo

Miyo is a mobile-first EPUB reader built with Expo and React Native. It focuses on comfortable long-form reading, web-novel terminology tools, offline-friendly typography, annotation workflows, and cloud backup.

## Current feature set

- EPUB import from device storage
- Watched-folder rescans on Android
- Duplicate import detection using file hash plus EPUB metadata
- Duplicate review flow with `Cancel import` or `Import anyway`
- Reader typography controls with bundled fonts
- Themes, layout controls, blue-light filter, bionic reading, auto-scroll, volume-button page turns
- Theme categories with `Normal` and `Special` themes
- Special-theme ambient art packs for Peach Blossom, Dark Coffee, Parchment Comfort, and Matcha Paper
- Highlights, bookmarks, annotation export, and reading stats
- Term groups and inline translation correction tools
- OPDS catalog browsing and direct import
- In-app WTR-LAB browsing with MTL search, description search, rank filters, chapter-range export, and EPUB download into the library
- Offline dictionary downloads with online fallback lookup
- Supabase-backed account system
- Beta mode currently disables Google sign-in and cloud sync on purpose

- Supabase Auth + Postgres

- AsyncStorage + SecureStore for local persistence

## Supabase setup

Client configuration is read from Expo public env variables in [/root/Epub-Reader-/lib/supabase.ts](/root/Epub-Reader-/lib/supabase.ts).

Create a local `.env.local` or set EAS env vars with:

```bash
EXPO_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
EXPO_PUBLIC_DISABLE_GOOGLE_SIGN_IN=1
EXPO_PUBLIC_DISABLE_CLOUD_SYNC=1
```

Google env vars are still optional future hooks, but the beta build keeps them off:

```bash
EXPO_PUBLIC_GOOGLE_DRIVE_CLIENT_ID=
EXPO_PUBLIC_GOOGLE_DRIVE_ANDROID_CLIENT_ID=
EXPO_PUBLIC_GOOGLE_DRIVE_IOS_CLIENT_ID=
EXPO_PUBLIC_GOOGLE_DRIVE_WEB_CLIENT_ID=
```

### 1. Database tables

Run [/root/Epub-Reader-/supabase-setup.sql](/root/Epub-Reader-/supabase-setup.sql) in the Supabase SQL editor.

This creates:

- `community_term_groups`
- `community_dictionaries`

It also seeds example community term groups and downloadable dictionary packs.

### 2. Auth URL configuration

In Supabase Auth settings, add the mobile redirect used by the app:

- `miyo://`
- `miyo://auth/confirm`

If you also test in Expo development environments, add the relevant development redirect URLs generated by Expo for your setup.

### 3. Email verification template

A more polished confirmation email template is included at:

- [/root/Epub-Reader-/supabase/email-templates/confirm-signup.html](/root/Epub-Reader-/supabase/email-templates/confirm-signup.html)
- [/root/Epub-Reader-/supabase/email-templates/confirm-signup-subject.txt](/root/Epub-Reader-/supabase/email-templates/confirm-signup-subject.txt)

Paste that HTML into the Supabase dashboard’s `Confirm signup` email template.
The template uses Supabase `RedirectTo` plus the app route `/auth/confirm`, so the email deep link returns to Miyo instead of a dead localhost page.

## Email verification flow

Verification links now return to an in-app confirmation screen at:

- [/root/Epub-Reader-/app/auth/confirm.tsx](/root/Epub-Reader-/app/auth/confirm.tsx)

If verification still opens a dead localhost page, the redirect URL configuration in Supabase is wrong. The confirmation template in [/root/Epub-Reader-/supabase/email-templates/confirm-signup.html](/root/Epub-Reader-/supabase/email-templates/confirm-signup.html) should point to `{{ .RedirectTo }}auth/confirm?...`, not localhost.

## Dictionaries

Offline dictionary downloads are available from Settings.

Behavior:

- If a matching offline dictionary entry exists, Miyo uses it first.
- If no offline match exists and the device is online, Miyo falls back to online lookup.
- If Supabase dictionary rows are unavailable, built-in local packs are still shown so the feature does not appear empty.

## WTR-LAB browser

Miyo now includes an in-app `Online MTL Browser` on the Library screen.

Current filters and tools:

- title/keyword search
- description contains search
- minimum and maximum total chapters
- minimum rating
- minimum review count
- latest toggle
- status filter
- updated/rating/chapter sorting
- load more pagination
- chapter-range EPUB export into the existing library import flow

Technical note:

- WTR-LAB uses Cloudflare and browser-only protection.
- Miyo handles this by running the parser in a WebView/browser context instead of a raw server fetch.
- The included live verification script exercises the same browser-context parser path:
  - [/root/Epub-Reader-/scripts/wtr-lab.live.test.ts](/root/Epub-Reader-/scripts/wtr-lab.live.test.ts)
- In headless CI, Cloudflare may still block automation. On-device WebView behavior is the actual target path for the app feature.

## OPDS catalogs

Miyo supports public OPDS feeds and includes a default Project Gutenberg catalog.

Notes:

- OPDS feeds must use public `https://` URLs.
- Local-network and insecure feeds are blocked.
- Requests time out instead of hanging indefinitely.
- Some OPDS feeds reject anonymous app requests and may return `401` or `403`.
- Those feeds are treated as unsupported/auth-required rather than as a generic app crash.

## Storage model

Miyo uses three different storage concepts:

- library storage: imported EPUB records tracked by the app
- watched folder: Android folder used for rescans/import discovery
- temporary cache: chapter cache and temporary downloads

`Clear Cache` now removes temporary cache only. It should not remove:

- bookmarks
- highlights
- reading stats
- reading position
- library entries

## Troubleshooting

### Fonts do not change in the reader

- Use a dev build or production build.
- Confirm the selected font is one of the bundled fonts in Settings.
- If the reader was already open during a code update, reload the app once.

### Google sign-in and cloud sync are unavailable

- This beta intentionally disables them with env flags.
- Leave `EXPO_PUBLIC_DISABLE_GOOGLE_SIGN_IN=1`.
- Leave `EXPO_PUBLIC_DISABLE_CLOUD_SYNC=1`.

### OPDS feed fails

- Try the default Project Gutenberg catalog first.
- Use a public `https://` feed, not a local address.
- If the feed returns `401` or `403`, it likely requires auth or blocks anonymous clients.
- The OPDS sheet now supports feed-level filtering plus previous/next page navigation when the feed exposes those links.

### Verification email opens but does not finish in the app

- Verify Supabase redirect URLs include `miyo://` and `miyo://auth/confirm`.
- Make sure the `Confirm signup` template uses `{{ .RedirectTo }}auth/confirm?...`.
- Open the link on the same device where Miyo is installed.

### Dictionary library is empty

- Run the SQL seed in Supabase.
- If Supabase is unavailable, check whether the built-in local packs still appear.

### WTR-LAB live test fails

- The bundled test uses headless Chromium.
- WTR-LAB may show a Cloudflare challenge in headless environments.
- That does not automatically mean the in-app Android WebView path is broken.
- Run the same feature on-device if the live script reports a browser challenge page.
